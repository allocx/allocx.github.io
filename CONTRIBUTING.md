# Contributing

## Git Commit Convention

Format: `<type>(<scope>): <subject>`

All commits must be:
- One line only (no body, no footer)
- Imperative tone ("add feature" not "added feature" or "adding feature")
- Lowercase subject, no period at the end

### Types
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `test`: Adding tests
- `refactor`: Code refactoring
- `chore`: Build/deps updates
- `perf`: Performance improvements
- `style`: Code formatting

### Scopes
- `blog`: Blog posts and content
- `config`: Jekyll configuration
- `assets`: CSS, images, favicons
- `seo`: SEO-related files (robots.txt, sitemap, meta)

### Examples
```
feat(blog): add post on memory management
fix(config): correct permalink format
docs(blog): update about page
chore(assets): optimize favicon sizes
refactor(config): simplify plugin list
```
